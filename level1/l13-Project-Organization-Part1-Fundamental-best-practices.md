# Lesson 13 - Project Organization Part 1 - Fundamental best practices

## 프로젝트 체계화

* 안전성
    + 모든 자동화에서 기대되는 특성
    + 자동화의 오류를 파악하고 그 영향을 최소화 하며 내외부에서 발생 가능한 오류를 해결할수 있어야 한다.
* 효율성
    + 개발 시간을 단축해야 한다.
    + 자동화가 운영환경에서 매끄럽게 실행되야 한다.
* 유지 관리
    + 워크플로우를 간편하게 유지관리 할수 있게 해야 한다.
* 확장성
    + 기존 자동화에 더욱 많은 것들이 통합 될수 있기 때문이다.

## best practices

* 프로젝트를 시작할때 적절한 레이아웃을 선택해야 한다.
* 대부분의 작업은 시퀀스나 플로우차트 모두가 가능하지만 실제 작업에 따른 환경을 선택 하는것이 중요하다.
    + 기본 워크 플로우는 플로우 차트를 사용하는게 더 좋다.
        + 전체 프로젝트를 개괄적으로 파악하기 더 쉽고 워크 플로우의 개별부분을 테스트하기도 더 쉽기 때문이다.
    + 비즈니스 로직 또한 플로우차트를 사용하는게 더 좋다.
    + UI들의 인터렉션은 차례대로 수행되고 큰 변동이 없기 떄문에 시퀀스가 적합하다.
+ 중첩 if는 가급적 사용을 자제 하자.
+ 자동화를 더 작은 컴포넌트 단위로 나누자.
+ 자동화 개발자 작업이 복잡성을 관리하는 일이라는 점.
    + 독립적인 컴포넌트의 기능을 개발하고 테스트하기 더 쉽다.
    + 동일 기능에 대한 재사용을 하여 개발시간이 단축 된다
    + 협업하기가 쉬워진다.
+ 시퀀스나 플로우차트 말고 Invoke 명령을 사용해도 좋다.
+ 예외처리를 사용해 오류를 포착하고 로깅하자.
+ 외부 워크플로우 파일을 사용할때 Try-Catch를 사용하자
+ 워크플로우를 읽기 쉽게 만들어라. 메모와 주석 그리고 로깅.
+ 사용하지 않는 애플리케이션은 끄자.
